# User Defined:

  • Stack

# List:

  • We can store different types of data in list. it can be int or string
  • List maintained its order
  • List can be access using indexing. ex: List = [0]
  • Negative indexing in list gives previous values of called index. ex: [-2]
  • List can be nested. means we declare different value in one list. ex: [[1,3], 1,4]
  • List are dynamic. Though, its mutable, value will shrinks after delete or add

# Tuple:

  • Similar to list which also store data
  • To Declare tuple value we have to use parentheses and for single value we have to use comma after the value. ex: tuple = (1,)
  • We can't change or modify tuple after creating it.We can't insert element neither delete any element or value
  • We use tuple because its faster than List and it can't be modified

# Dictionary:

  • Dictionary is for storing data using values with keys. key:value pair bounded with second bracket. ex: dic ={"x":1, "y":2}
  • Keys and values are spread by colon and each pair of keys and values spread by comma
  • keys must be constant but values can be change
  • From Python version 3.7, dictionaries are ordered but it was unordered in Python 3.6 and earlier
  • It can't be nested

# Sets:

  • Its a collection of unique elements. We can't have duplicate values in sets
  • If we use duplicate value it will ignore that value
  • Sets can contain only immutable types value
  • Sets can't be nested
